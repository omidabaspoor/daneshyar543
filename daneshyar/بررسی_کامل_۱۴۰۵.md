## ۱. خلاصه پروژه

**دانش‌یار** یک سامانه آموزشی هوشمند مبتنی بر وب است که با **PHP خام (بدون فریم‌ورک)** و **MySQL (PDO)** نوشته شده و با استفاده از یک مدل هوش مصنوعی چندوجهی (multimodal) به سوالات درسی دانش‌آموزان پایه ۷ تا ۱۲ پاسخ می‌دهد.

| مورد | مقدار |
|------|-------|
| زبان | PHP + JavaScript (وانیلا) + CSS |
| دیتابیس | MySQL از طریق PDO |
| سرویس AI | GapGPT API (`api.gapgpt.app`) |
| مدل تنظیم‌شده | `gemini-3-flash-preview` (در `config.php`) |
| فونت | Vazirmatn (لوکال) |
| کتابخانه‌های کلاینت | KaTeX + Marked.js (هر دو لوکال، بدون CDN) |
| تعداد فایل PHP | ۳۸ فایل (~۷۱۳۳ خط) |
| سازنده | امید عباسپور |

### نتیجه راستی‌آزمایی فنی (انجام‌شده در همین بررسی)
- ✅ **بررسی نحوی (lint) همه ۳۸ فایل PHP** → بدون هیچ خطای نحوی (`php -l`).
- ✅ **بررسی نحوی `assets/js/chat.js`** با Node → سالم.
- ✅ ساختار پوشه‌ها منظم و لایه‌بندی تمیز (config / db / functions / ai / views).

## ۳. نقاط قوت

1. **کیفیت کد بالا و خوانا** — نام‌گذاری مناسب، کامنت‌های فارسی، جداسازی صحیح لایه‌ها، استفاده‌ی یکدست از PDO با prepared statements و تابع `e()` برای escape خروجی HTML.
2. **مهندسی استریم (SSE) حرفه‌ای** در `api/chat.php`:
   - تشخیص JSON خطا در ابتدای استریم (وقتی API به‌جای SSE خطا می‌فرستد).
   - **محافظت در برابر loop بی‌نهایت مدل** (سقف ۳۰هزار کاراکتر).
   - **تشخیص خروجی خراب (garbage tail)** با چند heuristic هوشمند.
   - **Refund خودکار سهمیه** هنگام خطا یا قطع ارتباط کاربر (`connection_aborted`).
3. **سیستم اشتراک پیشرفته** — پلن‌های ساعتی/هفتگی/ماهانه، سهمیه روزانه و کلی، سهمیه رایگان روزانه، ریست خودکار، زمان‌بندی فعال‌سازی (`approved_pending`) و تخفیف اختصاصی هر کاربر.
4. **رزرو سهمیه به‌صورت atomic** (`reserve_message_quota`) برای جلوگیری از race condition.
5. **سازگاری کامل با اینترنت ایران** — همه vendorها لوکال، فونت لوکال، تبدیل تاریخ شمسی↔میلادی و ارقام فارسی↔انگلیسی.
6. **مدیریت چندوجهی ضمیمه** — تصویر نرمال‌سازی می‌شود (HEIC→JPG، resize)، و PDF به‌جای استخراج متن regex، مستقیماً base64 به مدل multimodal پاس می‌شود (تصمیم درست برای PDF فارسی).
7. **سیستم RAG** (`book_chunker.php`) — chunk کردن متن کتاب و انتخاب بخش‌های مرتبط با سوال.
8. **System Prompt بسیار دقیق** — تأکید بر خواندن نمادهای ریاضی، روش گام‌به‌گام، چک معکوس جواب و فرمول‌نویسی اجباری LaTeX.

## ۵. مشکلات فنی و معماری (🟠)

1. **Auto-migration در هر درخواست** — توابع `ensure_education_schema()`، `ensure_discounts_schema()`، `ensure_payment_schema()` و چند `CREATE/ALTER/UPDATE` در **هر بار بارگذاری صفحه** اجرا می‌شوند (انتهای `functions.php`). این باعث کندی و احتمال database lock زیر بار بالا می‌شود. (هرچند با `static $done` در همان request یک‌بار اجرا می‌شوند، ولی در هر request جدید دوباره `SHOW COLUMNS`/`CREATE IF NOT EXISTS` می‌زنند.) → باید فقط در install/update اجرا شوند.
2. **نبود Rate Limiting واقعی** — محدودیت فقط بر اساس شمارنده‌ی سهمیه است؛ محدودیت بر اساس IP یا فاصله‌ی زمانی بین درخواست‌ها وجود ندارد (به‌ویژه در `api/upload.php`).
3. **`install.php` و `migrate.php` در ریشه** — اگر بعد از نصب حذف نشوند، خطر re-initialize شدن دیتابیس وجود دارد. قفل خودکار ندارند.
4. **فایل‌های موقت توسعه در ریپو** — `fix_*.py`, `patch_*.py`, `add_contact_btn.py`, `fix_functions.php`, `patch_api.php`, `debug_anim.html` و `preview*.html` نباید در نسخه‌ی production باشند.
5. **نبود Foreign Key** در اغلب جداول (فقط `user_discounts` دارد) → امکان باقی‌ماندن رکوردهای orphan.
6. **مدل ناسازگار با مستندات** — در `config.php` مدل `gemini-3-flash-preview` است ولی در `REVIEW_REPORT.md`/README به `gemini-2.5-flash` اشاره شده. باید یکدست شود.
7. **تابع `pdf_extract_text` با regex** هنوز در `functions.php` باقی مانده (هرچند مسیر اصلی چت دیگر از آن استفاده نمی‌کند) — کد مرده.

## ۶. پیشنهادهای بهبود (اولویت‌بندی‌شده)

### 🔴 فوری (امنیت)
1. کلید API لو رفته را revoke و کلید جدید را به **متغیر محیطی / فایل `.env`** منتقل کن.
2. یک `.gitignore` بساز و `includes/config.php`، `uploads/`, `books/*.pdf` را از ردیابی خارج کن؛ یک `config.sample.php` بدون مقادیر حساس قرار بده.
3. `DEV_MODE` را در production روی `false` بگذار.
4. رمز ادمین و اطلاعات کارت/تماس را از کد خارج کن (DB یا `.env`).
5. رسیدهای پرداخت واقعی را از تاریخچه‌ی git پاک کن (`git filter-repo`).

### 🟠 کوتاه‌مدت (پایداری)
6. Auto-migration را به یک اسکریپت جداگانه‌ی `migrate.php` منتقل کن که فقط دستی اجرا شود.
7. `SSL_VERIFYPEER` را فعال و یک CA bundle معتبر تنظیم کن.
8. Rate limiting بر اساس IP/زمان روی `chat.php` و `upload.php` اضافه کن.
9. فایل‌های `fix_*`/`patch_*`/`debug_*`/`preview_*` و `install.php` را از production حذف کن.

### 🟢 بلندمدت (محصول)
10. درگاه پرداخت آنلاین (زرین‌پال/IDPay) به‌جای فقط کارت‌به‌کارت.
11. Foreign Key و ایندکس‌گذاری بهینه در اسکیمای SQL.
12. تبدیل به PWA برای نصب روی موبایل.
13. لاگ خطاها در فایل به‌جای نمایش به کاربر.
14. CSP header و توکن CSRF per-request.

## ۷. جمع‌بندی

دانش‌یار یک **پروژه‌ی فول‌استک PHP سطح متوسط رو به حرفه‌ای** است با کدنویسی تمیز، مدیریت استریم AI بسیار خوب، و سازگاری کامل با شرایط ایران. **منطق برنامه و کیفیت کد قوی است** و همه‌ی فایل‌ها از نظر نحوی سالم‌اند.

اما برای انتشار عمومی، **چند مشکل امنیتی بحرانی** وجود دارد — مهم‌ترین آن‌ها لو رفتن **کلید API، رمز ادمین و اطلاعات کارت بانکی** در یک مخزن عمومی است که باید **فوراً** برطرف شوند. با رفع موارد امنیتی و چند بهبود معماری (به‌ویژه حذف auto-migration از مسیر درخواست)، این پروژه به یک محصول قابل‌ارائه تبدیل می‌شود.
